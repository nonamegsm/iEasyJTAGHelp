Apple Device Guide

1. Theory

The most convenient way to replace a memory card in Apple devices is to replace it with iTunes. Using official software avoids unnecessary errors and installs the original firmware in the refurbished device.

The only problem in this case is copying the serial data needed to further activate and use the device. To successfully copy and record serial data, you need to remember the following features:

  • 1) Each generation of devices has a clear list of usable memory cards. Before selecting a card, make sure it's right for the device.
2) Each generation of devices for each memory card model has a unique storage unit for serial information. Thus, you can only fully copy a block with serial information on a map of the exact same model.
  • 3) Devices are separated by discharge by generation:
  1. (a) Devicesx32: - iPad2 - iPad3 - iPad4 - iPhone4s - iPhone5 - iPhone5c - mini1
  2. b) Devices x64: - iPad5 - iPad6 - iPadAir - iPadAir2 - iPhone5s - iPhone6 - iPhone6P - mini2 - mini3 - mini4
  • 4) Devices have different storage patterns. Thus, if you move a map from one model to another, you may need pre-formatting. You can format the map with iTunes, erase it completely enough and restore it with iTunes. Also, this action may be necessary when using a new (clean) card.
  • 5) When you restore your device with iTunes, auto-rotation information is copied to iOS, where it is stored encrypted and where it will be read the next time you use iTunes. So if you need to change the auto-rotational information, you need to not only change it in the auto-rotation block, but also remove it from the OS. The

2. Algorithm

  • 1) Remove the memory card from the device and read from it auto-rotation allotoninformation. Or get information in any other way.
  • 2) Install a new memory card in your device and format (restore) it with iTunes before you erase it.
  • 3) Remove the memory card and write it auto-rotation alloton information. Not forgetting to delete OS data.
  • 4) Restore your device with iTunes.

 

3. Apple's Auto-Rotation Information Interface (later iConfig) in EasyJtag Classic


The suite has created a separate tab. To choose from, go to the NAND toolbar tab and then select the iConfig tab.

The working area can be divided into four parts:

  • 1) NAND General is a global panel for all NAND tabs, allowing you to identify the memory card, as well as adjust the frequency and voltage. Error management options don't affect iConfig.
2) IConfig's display and settings area. This area displays Apple's auto-rotation al-data for this map or settings file. Please note that identification does not change information in these fields.

The information entered into the field data will be used on the recording and will replace the information in the iConfig memory card. If you don't want to change values in a field, just leave it empty.

It should also be noted that this panel stores all possible options iConfig. As mentioned above, the same memory card model can be used in different devices, and as a result, can have different ways of storing iConfig. To view all iConfig, use the Drop list mode in this area. Attention! Mode in this area only switches information about iConfig, but does not affect its record.

  • 3) The iConfig reading and selection area allows you to read the data (Read iConfig button) and save the areas with them (Read and save iConfig area). Saving takes place in master iConfig file, which automatically opens the program after the save.

  1. In this area, you can select files with area data (iConfig area) that will be recorded, as well as open the previously saved Master iConfig file.

  • 4)  The iConfig recording area lets you set up Write Mode. Most often, the cards have one recording mode for each discharge (x32 or x64). However, sometimes there are cards with two recording modes for each of the discharges. To choose the right one, just read the formatted (restored) map with iTunes and see in which case a more complete block of data has returned.

The Erase data option includes a complete cleaning (erasure) of the card before recording or stering the iConfig area.

The Fix iConfig button changes the iConfig data recorded in the map, but doesn't change the rest of the data in the storage area (iConfig area). Convenient to work with a map that's already been restored with iTunes.

The Erase all iConfig areas button removes everything you can for this iConfig area. Lets you start the iConfig area recovery process with iTunes.

The Write iConfig area from files is completely rewrites iConfig area data and replaces it with file data listed in the previous area. When you write, iConfig data is replaced in the iConfig area in the same way as Fix iConfig. It can be useful when cloning a map or if necessary to record your data. It is worth recalling that the iConfig area is interchangeable only for the same map model and only within one generation of devices.

The Write iConfig for Selected Device button maps the iConfig area we've prepared for the Device device we select in the field and the map we use. This changes iConfig

4. Standard software algorithm

  • 1) Remove an old memory card from your device.
  • 2) Identify it (Identify NAND Chip button).
  • 3) Read and save the iConfig area (Read and save iConfig area).
  • 4) Identify a new card (Identify NAND Chip button).
  • 5) Read iConfig (Read iConfig button). If a new map is taken from a device on the same model, we go straight to p.14.
  • 6) Erase data from the card (Erase all iConfig areas button with Erase data enabled).
  • 7) Restore iConfig area with iTunes (install map into device and restore when iTunes).
  • 8) Remove the map and read iConfig (Read iConfig button). If iConfig is successfully read, let's move on to p.14.
  • 9) Fill iConfig fields manually or open Master iConfig file (saved in p.3).
  • 10) Record, prepared by us, iConfig area. To do this, you need to choose a recording mode (Write Mode), select a device ,Device), set the Erase data option and click Write iConfig for Selected Device.
  • 11) Restore to install using iTunes (install map into device and run iTunes).
  • 12) Check the device after recovery, in case the data is recovered correctly moving on to p.18
  • 13) Remove the map and read iConfig (Read iConfig button). In the case of iConfig reading, we move on to p.14. 
  • 14) Fill iConfig fields manually or open Master iConfig file (saved in p.3).
  • 15) Change iConfig data. To do this, you need to choose a write mode (Write Mode), set Erase data and click iConfig.
  • 16) Install a map and start the recovery using iTunes.
  • 17) Check the device after recovery. If the recovery went wrong, a different non-standard solution is needed. Save the work log and Master iConfig file and get in touch to further address this issue.
  • 18) Enjoy your work device!

You can also use a simplified algorithm to just skip the items from 6 to 8

Inclusive. This will save time and reduce the risks of damaging the device while installing and removing the memory card. But, in some non-standard cases, you need to use iConfig area restoration with iTunes (p.6-8).
